# Purpose

Create a small LRS ontology for the purpose of RSM.

# Rationale

A successor to a part of the "Positioning" package in RTM. The former Positioning package covered:

* geographic or geometric referencing systems
* linear referencing systems

The geo-... referencing systems are now managed by using GeoSPARQL, and in the future by using IFC (esp. for local coordinate systems).

The linear referencing systems need to be addressed here. They are also defined in IFC, which adheres with ISO19148 to a large extent. However at this stage (end of MOTIONAL project) we need a simplified, self-standing representation. 

Semantic links with IFC are obviously welcome, especially w.r.t. future evolutions.
